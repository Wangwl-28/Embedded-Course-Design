# 基于STM32F103C8T6的智能手表设计  
## 项目验收报告


# 一、项目设计目标

## 1.1 总体目标

设计并实现一款基于 STM32F103C8T6 的智能手表原型系统，实现以下功能：

1. 完成智能手表硬件平台搭建；
2. 实现 OLED 显示界面；
3. 实现 MPU6050 运动传感数据采集；
4. 基于 FreeRTOS 实现多任务调度；
5. 实现人体运动状态检测；
6. 设计计步算法，实现步数统计功能。

# 三、硬件设计


## 3.1 主控模块

系统采用 STM32F103C8T6 作为核心控制器。

主要特点：

- ARM Cortex-M3内核；
- 主频72MHz；
- 丰富GPIO资源；
- 支持I2C、USART等通信接口；
- 满足智能手表实时控制需求。


---

## 3.2 OLED显示模块

系统采用I2C接口OLED显示屏作为人机交互界面。

连接方式：

|OLED|STM32|
|-|-|
|VCC|3.3V|
|GND|GND|
|SCL|PB6|
|SDA|PB7|

OLED主要用于显示：

- 当前时间；
- 运动数据；
- 计步结果。


---

## 3.3 MPU6050运动检测模块

系统采用 MPU6050 六轴运动传感器采集人体运动信息。

本项目主要使用 MPU6050 三轴加速度计功能。

连接方式：

|MPU6050|STM32|
|-|-|
|VCC|3.3V|
|GND|GND|
|SCL|PB6|
|SDA|PB7|


传感器通过I2C总线向STM32发送加速度数据，为运动检测和计步算法提供数据基础。


---

# 五、计步算法设计


## 5.1 算法原理

人体行走过程中，手腕运动会导致三轴加速度产生周期性变化。

系统通过MPU6050采集：

\[
a_x,a_y,a_z
\]


计算合成加速度：

\[
A=\sqrt{a_x^2+a_y^2+a_z^2}
\]


然后通过滤波和峰值检测判断步态变化。

# 六、功能测试与结果

## 计步功能测试


测试方式：

实际佩戴智能手表进行步行测试。


测试结果：

|项目|结果|
|-|-|
|实际行走步数|18步|
|系统检测步数|22步|
|误差|4步|
|误差率|22.2%|


分析：

测试结果表明系统能够识别人类行走过程中的周期性运动变化，实现基本计步功能。

误差主要来源包括：

1. 手腕摆动产生额外加速度峰值；
2. 行走过程中动作幅度变化；
3. 传感器噪声影响。


后续可以通过优化滤波算法、增加动态模型以及多传感器融合方式进一步提高准确率。
