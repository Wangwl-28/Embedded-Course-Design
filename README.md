# 《项目计划书》
—— 基于STM32F103C8T6智能手表的设计

## 一、工作计划
### 1.1 总体时间规划
| 阶段 | 时间 | 主要任务 |
|------|------|----------|
| 第一阶段 | 2026.06.29 | 选题与方案论证（基于开源项目分析） |
| 第二阶段 | 2026.06.29-06.30 | 硬件设计与器件选型 |
| 第三阶段 | 2026.07.01-07.02 | 工程搭建与驱动框架建立 |
| 第四阶段 | 2026.07.03-07.06 | UI系统与外设联调 |
| 第五阶段 | 2026.07.07-07.09 | 功能扩展与系统优化 |
| 第六阶段 | 2026.07.10 | 最终测试与验收 |


### 1.2 详细工作计划
第一阶段：选题与方案论证（6月29日）
- 分析 STM32SmartWatch 项目结构与实现逻辑
- 明确系统核心架构（UI状态机 + 驱动层 + 数据层）
- 确定软件架构
- 完成系统功能定义与模块划分

第二阶段：硬件设计与物料准备（6月29日-6月30日）
- 绘制系统原理图：
1. STM32最小系统
2. OLED（I2C）
3. MPU6050（I2C）
4. 蓝牙模块（USART）
5. 电源与锂电池管理模块
- 采购核心器件并确认到货

第三阶段：工程搭建与驱动开发（7月1日-7月2日）
- 搭建STM32基础工程（HAL/标准库均可）
- 完成外设驱动开发：
1. OLED驱动
2. MPU6050驱动
3. 蓝牙串口驱动
- 构建基础UI状态机框架
- 实现基础页面切换逻辑

第四阶段：UI系统与联调（7月3日-7月6日）
- 实现多级菜单UI系统（状态机驱动）
- 实现页面功能：
1. 时间显示页面
2. 传感器数据页面
3. 蓝牙通信页面
4. 设置页面
- 完成编码器菜单控制逻辑
- 实现I2C/UART稳定通信
- 可选引入FreeRTOS进行任务划分
- 完成中期检查报告

第五阶段：功能扩展与优化（7月7日-7月9日）
- 扩展功能1：蓝牙手机数据同步
- 扩展功能2：UI交互优化（菜单流畅性提升）
- 扩展功能3：PCB设计或低功耗优化（二选一）
- 可选扩展4：计步功能实现（基于加速度阈值）
- 系统稳定性测试与Bug修复

第六阶段：验收与总结（7月10日）
- 完成功能整体验收测试
- 优化UI显示与系统稳定性
- 准备答辩演示材料
- 完成项目总结报告
